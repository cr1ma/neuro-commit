# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

We recommend always using the latest version of **neuro-commit** to ensure you have the most recent security fixes.

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, open a **[private security advisory](https://github.com/cr1ma/neuro-commit/security/advisories/new)** on GitHub

### What to Include

When reporting a vulnerability, please provide:

1. A description of the vulnerability
2. Steps to reproduce the issue
3. Potential impact of the vulnerability
4. Any suggested fixes (if applicable)

## Dependency Security

We use `npm audit` to regularly check for known vulnerabilities in our dependencies. If you notice a vulnerable dependency, please open an issue or submit a pull request updating it.
